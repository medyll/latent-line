# ST-023 — PropertiesPanel: character asset inline editing

- sprint: 6
- created: 20260308T
- status: ready
- points: 5

## Description

Enable inline editing of character assets in PropertiesPanel. Users can click a character asset field to reveal a dropdown selector with available characters, or inline-edit character properties (name, emotion, mood).

## Acceptance Criteria

- Character field renders as editable chip with dropdown icon
- Dropdown lists all available characters from `model.assets.characters[]`
- Selecting a character from dropdown updates `model.timeline[idx].character`
- Inline edit mode: double-click character name → text input, blur/Enter saves
- Mood selector dropdown (existing) works in character section
- Unit tests: 5 tests (dropdown render, selection, inline edit, mood sync)
- No visual regression in existing PropertiesPanel fields (PP-01, PP-02, PP-06–10)

## Implementation Notes

- `PropertiesPanel.svelte`: add `<CharacterField>` component to event editing section
- New component: `src/lib/components/app/CharacterField.svelte`
  - Accepts `character: string`, `characters: {id, name, emotion, mood}[]`
  - Two-way bind via parent-dispatched events: `on:change={e => model.timeline[idx].character = e.detail}`
- Dropdown: `<select bind:value={character}>` or custom select if shadcn has one
- Inline edit: `<input bind:value={editingName} on:blur={save}>` with focus trap
- Mood selector: read from `assets.characters[id].mood` and persist to model
- Test: `PropertiesPanel-st023.test.ts` — 5 tests for character field states
