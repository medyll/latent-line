# ST-024 — Audio timeline lanes / multi-track view

- sprint: 6
- created: 20260308T
- status: ready
- points: 8

## Description

Add multi-track audio timeline view to SequenceOrchestrator. Display audio assets in horizontal lanes below the main event sequencer. Each lane represents one audio track; events can be dragged between lanes. Support play/solo/mute per lane.

## Acceptance Criteria

- Audio track lanes rendered below Temporal Sequencer
- Each lane: mute/solo buttons (left sidebar), waveform placeholder (center), mute/solo state persisted to `model.config.audioLanes[id]`
- Drag audio events between lanes (lane indicator shows on hover)
- Play button respects mute/solo state (solo = only that track plays)
- Keyboard shortcut: `M` to mute selected lane, `S` to solo
- Unit tests: 8 tests (lane render, drag between lanes, mute/solo logic)
- E2E test: add audio track, drag event, toggle solo (1 test)
- No regression in existing Temporal Sequencer (ST-021)

## Implementation Notes

- New component: `src/lib/components/app/AudioTimeline.svelte`
  - Accepts `model: Model`, `audioTracks: AudioTrack[]`
  - Lane structure: `{id, name, muted, soloed, events: TimelineEvent[]}`
- Each lane: `<div class="audio-lane">` with absolute-positioned audio events
- Mute/Solo: toggle `model.config.audioLanes[id].muted|soloed` and re-render
- Drag: same collision logic as Temporal Sequencer (ST-021); resolve lane + time
- Play logic in SystemFooter: check `model.config.audioLanes[].soloed` before playing
- Waveform: placeholder `<div class="waveform-placeholder">` (actual waveform rendering deferred)
- Test: `AudioTimeline.test.ts` — 8 tests for lanes, drag, mute/solo
