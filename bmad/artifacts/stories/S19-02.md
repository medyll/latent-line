# S19-02 — Contrastes WCAG AA + ARIA exhaustifs

**Sprint:** 19
**Points:** 3
**Priorité:** 🟡 Moyenne

## Contexte

Plusieurs éléments de l'UI ont des ratios de contraste insuffisants (texte gris sur fond clair). Les ARIA labels ne couvrent pas tous les contrôles dynamiques.

## Objectif

Atteindre WCAG 2.1 AA sur l'ensemble de l'application (ratio ≥ 4.5:1 pour le texte normal).

## Critères d'acceptance

- [ ] Audit axe-core : 0 violations de niveau AA
- [ ] Tous les inputs ont un `<label>` ou `aria-label`
- [ ] Tous les boutons icônes ont `aria-label` descriptif
- [ ] Les états dynamiques (selected, expanded, loading) sont annoncés via `aria-*`
- [ ] Live regions (`aria-live="polite"`) pour les notifications
- [ ] Contraste vérifié en mode clair ET sombre

## Implémentation suggérée

- `npm install @axe-core/playwright` pour les tests E2E
- Revue des tokens CSS : ajuster `--color-text-muted`, `--color-border`
- `e2e/accessibility.spec.ts` — test axe sur chaque route principale
