# S20-02 — Code splitting et lazy loading

**Sprint:** 20
**Points:** 3
**Priorité:** 🟡 Moyenne

## Contexte

Le bundle actuel charge tous les composants au démarrage, y compris ModelInspector et PreferencesPanel qui sont rarement utilisés.

## Objectif

Réduire le bundle initial via le code splitting SvelteKit et le lazy loading des composants lourds.

## Critères d'acceptance

- [ ] Bundle initial < 150KB (gzip) mesuré via `vite build --analyze`
- [ ] ModelInspector chargé en lazy (import dynamique `() => import(...)`)
- [ ] PreferencesPanel chargé en lazy
- [ ] ExportModal chargé en lazy
- [ ] Temps de chargement initial < 1s sur réseau 3G simulé
- [ ] Rapport bundle inclus dans CI comme artefact

## Implémentation suggérée

- `{#await import('./ModelInspector.svelte') then {default: Cmp}} <Cmp/> {/await}`
- `vite-bundle-visualizer` en devDependency
- Script CI : `npm run build && npx vite-bundle-visualizer`
