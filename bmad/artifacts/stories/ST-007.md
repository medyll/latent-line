# ST-007 — PropertiesPanel: Full Asset Data

**Epic:** Shared State
**Sprint:** 2
**Points:** 2
**Priority:** Must
**Status:** Done

## User Story

As a user, I want to see the full details of a selected asset (name, prompt, voice, etc.) in the PropertiesPanel, so that I can inspect any asset without opening the AssetManager.

## Context

ST-003 added asset selection awareness to PropertiesPanel but could only show `type:id` — no actual data. With ST-006 providing shared context, PropertiesPanel can now look up and display the full asset record.

## Acceptance Criteria

```gherkin
Given a character is selected in AssetManager
When PropertiesPanel renders
Then it shows character name, id, voice_id (if any), outfit names

Given an environment is selected
When PropertiesPanel renders
Then it shows environment id and prompt text

Given an audio asset is selected
When PropertiesPanel renders
Then it shows audio id, label, and url

Given no asset is selected and no event is selected
When PropertiesPanel renders
Then it shows the empty state message
```

## Technical Notes

- Read assetStore from context (ST-006 prerequisite)
- Parse `selectedAssetId` (`type:id`) to determine asset type and look up record
- Display structured fields (not raw JSON) — use consistent `<dl>/<dt>/<dd>` or labeled rows
- Characters: show outfits as a collapsible list or badge count

## Out of Scope

- Editing asset properties from PropertiesPanel (deferred to Sprint 3)

## Dependencies

- ST-006 (assetStore in context)

## Definition of Done

- [x] PropertiesPanel shows full character data when char selected (name, voice_id, outfits, refs)
- [x] PropertiesPanel shows full environment data when env selected (id, prompt, ref)
- [x] PropertiesPanel shows full audio data when audio selected (id, label, url)
- [x] Structured display using `<dl>/<dt>/<dd>` — no raw JSON for assets
- [x] Unit test for asset lookup logic (9 tests in PropertiesPanel.test.ts)
