# VR-001 — Migrate visual regression to Playwright snapshot diffing

**Epic:** Visual Regression / Testing
**Sprint:** 8
**Points:** 2
**Priority:** Should

## User Story

As a **developer**, I want visual regression tests to use Playwright's built-in snapshot diffing, so that visual changes are detected automatically without a custom comparison script.

## Context

Sprint 7 (TST-002) set up visual regression using a custom `scripts/visual-regression.mjs` script with manual pixel comparison. Playwright has native `expect(page).toHaveScreenshot()` which handles baseline storage, comparison, and reporting natively. This story migrates to the standard approach.

## Acceptance Criteria

```gherkin
Given the visual tests run via `npm run test:e2e`
When a component has not changed visually
Then the snapshot test passes without manual baseline comparison

Given a component's visual appearance changes
When snapshot tests run
Then the diff is automatically reported in the Playwright HTML report

Given CI runs the visual tests
When snapshots don't exist yet
Then CI generates baselines on first run and commits them

Given a developer updates a component intentionally
When they run `npx playwright test --update-snapshots`
Then the baseline is updated for the changed component only
```

## Technical Notes

- Migrate `e2e/visual-capture.spec.ts` to use `expect(page).toHaveScreenshot('ComponentName.png')`
- Remove dependency on `scripts/visual-regression.mjs` for diff logic (keep as utility if needed)
- Playwright stores snapshots in `e2e/__snapshots__/` by default
- Add `--update-snapshots` note to README and SCREENSHOTS.md
- Update `visual-regression.yml` to use `npx playwright test e2e/visual-capture.spec.ts` natively
- Platform note: add `snapshotPathTemplate` config to isolate Linux CI snapshots from Windows dev snapshots

## Out of Scope

- Generating new component baselines (manual step post-migration)
- Dark mode snapshot variants

## Dependencies

- E2E-001 (parallel — can be done independently)

## Implementation Notes

**Date:** 2026-03-09
**Files changed:**
- `e2e/visual-capture.spec.ts` — fully migrated to `toHaveScreenshot()` API (5 snapshot tests)
- `playwright.config.ts` — added `snapshotPathTemplate` for OS isolation
- `.github/workflows/visual-regression.yml` — simplified to use Playwright native runner

**Notable decisions:**
- Used `snapshotPathTemplate` with `{platform}` token to isolate Linux/Windows baselines
- Reduced test count from 10 to 5 focused snapshots (full-page, AssetManager, PropertiesPanel states)
- Removed custom `scripts/visual-regression.mjs` from CI path (script retained for local use)
- `continue-on-error: true` retained so CI doesn't block PRs when baselines don't exist yet

**Known limitations:**
- Baselines will be generated on first CI run; subsequent runs will compare
- Components without `data-test-id` or `aria-label` skipped from snapshots (graceful skip)
