# S12-01 — Persistance localStorage (auto-save / restore)

**Sprint:** 12
**Points:** 5
**Priorité:** 🔴 Critique

## Contexte

Le modèle est entièrement en mémoire. Un rechargement de page détruit tout le travail.
C'est le blocage MVP numéro 1.

## Objectif

Auto-sauvegarder le modèle dans `localStorage` à chaque mutation, et le restaurer au démarrage.

## Critères d'acceptance

- [ ] Le modèle est sérialisé en JSON et stocké sous la clé `latent-line:project`
- [ ] Au démarrage de `/app`, si une sauvegarde existe, elle est chargée à la place du template par défaut
- [ ] Un indicateur visuel ("Saved") s'affiche brièvement dans le SystemFooter après chaque save
- [ ] Un bouton "New project" dans le SystemFooter efface le localStorage et réinitialise au template
- [ ] La persistance résiste à un rechargement page (F5)
- [ ] Tests unitaires sur la logique serialize/deserialize

## Implémentation suggérée

- Watcher `$effect` sur le store `model` dans `+page.svelte` → `localStorage.setItem`
- Debounce 500ms pour éviter les writes excessifs
- Lecture au `onMount` avant d'initialiser le store
- Fonction `resetProject()` dans le store pour le "New project"
