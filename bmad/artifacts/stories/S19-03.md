# S19-03 — Tests d'accessibilité automatisés

**Sprint:** 19
**Points:** 3
**Priorité:** 🟡 Moyenne

## Contexte

Sans tests automatisés, les régressions d'accessibilité passent inaperçues lors des refactos.

## Objectif

Intégrer axe-core dans le pipeline CI pour détecter les violations automatiquement.

## Critères d'acceptance

- [ ] `e2e/accessibility.spec.ts` avec axe-core sur : `/app`, `/timeline`, modales
- [ ] Seuil : 0 violations critiques + 0 violations sérieuses
- [ ] Rapport axe inclus dans les artefacts CI (`accessibility-report.json`)
- [ ] Badge accessibilité dans le README
- [ ] Tests passent en < 30s

## Implémentation suggérée

- `@axe-core/playwright` comme devDependency
- Helper `checkA11y(page, selector?)` dans `e2e/helpers.ts`
- Ajout dans `.github/workflows/ci.yml`
