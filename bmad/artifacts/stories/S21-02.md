# S21-02 — États vides et indicateurs de chargement

**Sprint:** 21
**Points:** 2
**Priorité:** 🟡 Moyenne

## Contexte

Les listes vides affichent juste "No assets" sans contexte. Il n'y a pas d'état de chargement lors de l'import d'un gros fichier JSON.

## Objectif

Améliorer tous les états vides et ajouter des indicateurs de chargement cohérents.

## Critères d'acceptance

- [ ] État vide de chaque liste : illustration SVG + texte d'action ("Ajoutez un personnage →")
- [ ] Spinner ou progress bar lors de l'import JSON
- [ ] Skeleton loader pour la liste d'events au démarrage
- [ ] Toast notifications pour les actions (sauvegardé, importé, erreur)
- [ ] `ToastManager.svelte` réutilisable dans toute l'app

## Implémentation suggérée

- `src/lib/components/ui/EmptyState.svelte` — composant générique
- `src/lib/components/ui/Toast.svelte` + `toastStore`
- `src/lib/components/ui/Spinner.svelte`
