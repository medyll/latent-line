# S20-03 — Optimisation des sauvegardes localStorage

**Sprint:** 20
**Points:** 2
**Priorité:** 🟡 Moyenne

## Contexte

La sauvegarde localStorage se déclenche à chaque frappe clavier lors de l'édition d'un champ. Sur les gros modèles, `JSON.stringify` prend plusieurs ms et bloque le thread.

## Objectif

Passer à un debounce intelligent et une sérialisation asynchrone.

## Critères d'acceptance

- [ ] Debounce de 500ms sur la sauvegarde (pas de save pendant la frappe active)
- [ ] Sauvegarde forcée immédiate avant `beforeunload` (perte de données = 0)
- [ ] Indicateur visuel "Sauvegardé" / "Modifications non sauvegardées" dans la topbar
- [ ] Mesure : temps de sauvegarde < 5ms pour un modèle de 100 events
- [ ] Tests unitaires sur le debounce

## Implémentation suggérée

- `src/lib/utils/persistence.ts` — ajouter `debouncedSave(model, 500ms)`
- `SaveIndicator.svelte` — composant topbar avec icône ✓ / ●
- `window.addEventListener('beforeunload', ...)` pour flush immédiat
