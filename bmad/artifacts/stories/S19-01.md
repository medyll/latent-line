# S19-01 — Navigation clavier complète

**Sprint:** 19
**Points:** 5
**Priorité:** 🟠 Haute

## Contexte

L'application est partiellement utilisable au clavier mais sans ordre de focus cohérent ni raccourcis documentés. Requis pour WCAG 2.1 AA.

## Objectif

Garantir que toutes les fonctions de l'app sont accessibles uniquement au clavier.

## Critères d'acceptance

- [ ] Tab order logique dans chaque panneau (de haut en bas, gauche à droite)
- [ ] Focus visible sur tous les éléments interactifs (outline CSS `2px solid var(--color-focus)`)
- [ ] Modale/popover : focus trap (Tab reste dans la modale, Escape ferme)
- [ ] Timeline : navigation avec ←/→ entre events, Enter pour sélectionner
- [ ] AssetManager : ↑/↓ dans les listes, Enter pour éditer, Delete pour supprimer
- [ ] Raccourcis documentés dans un panneau "?" (touche ?)
- [ ] Tests Playwright pour les principaux flux clavier

## Implémentation suggérée

- Action `use:focusTrap` dans `src/lib/actions/focus-trap.ts`
- `KeyboardHelpPanel.svelte` — overlay avec tableau des raccourcis
- Revue systématique `tabindex` dans tous les composants
