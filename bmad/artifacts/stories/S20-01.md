# S20-01 — Virtualisation de la liste d'events

**Sprint:** 20
**Points:** 8
**Priorité:** 🟡 Moyenne

## Contexte

Au-delà de 50 events, la synoptic view devient lente (tous les DOM nodes rendus simultanément). Les projets de long-métrage peuvent avoir 200+ events.

## Objectif

Implémenter un rendu virtualisé pour la synoptic view afin de ne rendre que les events visibles.

## Critères d'acceptance

- [ ] 200 events rendus sans dégradation de performance (< 16ms par frame)
- [ ] Scroll fluide avec overscan de 3 items au-dessus et en-dessous
- [ ] La sélection, l'édition et le drag & drop fonctionnent normalement
- [ ] Pas de dépendance externe (implémentation native)
- [ ] Tests de performance : mesure du temps de rendu avec 200 events

## Implémentation suggérée

- `src/lib/components/app/VirtualList.svelte` — composant générique
- Pattern : `scrollTop` → calcul de `startIndex/endIndex` → slice du tableau
- `IntersectionObserver` pour la détection fine
