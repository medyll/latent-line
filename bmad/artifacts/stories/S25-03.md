# S25-03 — Mode présentation (kiosk / review)

**Sprint:** 25
**Points:** 5
**Priorité:** 🟡 Moyenne

## Contexte

Lors des revues de projet avec les clients ou l'équipe, l'utilisateur veut présenter la séquence de façon lisible sans que l'interface d'édition ne distraie.

## Objectif

Ajouter un mode "Présentation" qui affiche la séquence en plein écran de façon épurée.

## Critères d'acceptance

- [ ] Touche `P` ou bouton "Présenter" → passe en mode présentation plein écran
- [ ] Affichage : une card par event, navigation ←/→ ou clic pour avancer
- [ ] Chaque card : numéro, personnage, mood (couleur), action, image générée si disponible
- [ ] Lecture automatique avec durée configurable par event
- [ ] Escape ou touche `P` → retour à l'édition
- [ ] Fonctionne sans souris (clavier uniquement)

## Implémentation suggérée

- `PresentationMode.svelte` — composant fullscreen avec `Fullscreen API`
- Route dédiée `/present` accessible via URL (partageable pour review)
- Lecture auto : `setInterval` basé sur la durée de chaque event
