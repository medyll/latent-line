# CI-001 — Main GitHub Actions pipeline (lint + typecheck + unit + E2E)

**Epic:** CI / Automation
**Sprint:** 8
**Points:** 3
**Priority:** Must

## User Story

As a **developer**, I want a GitHub Actions CI pipeline that runs on every push and pull request, so that regressions are caught automatically before merging.

## Context

The PRD milestone explicitly requires "Add CI to run lint/typecheck/unit/e2e (story: add-ci)". A `visual-regression.yml` workflow already exists but covers only visual tests. A main `ci.yml` is missing — this story delivers it.

## Acceptance Criteria

```gherkin
Given a push or pull request to any branch
When GitHub Actions triggers
Then the ci.yml workflow runs lint, typecheck, unit tests, and E2E tests in sequence

Given the unit test step runs
When all 218+ unit tests pass
Then the step exits 0 and the pipeline continues

Given lint or typecheck fails
When a developer pushes code with type errors or formatting issues
Then the CI pipeline fails fast and reports the error

Given an E2E test step runs
When Playwright tests execute against the built app
Then all E2E tests pass or failures are clearly reported

Given CI is green
When a developer opens a PR
Then the PR shows a green status check from "CI"
```

## Technical Notes

- Create `.github/workflows/ci.yml`
- Steps: checkout → setup-node (v20) → npm ci → npm run lint → npm run check → npm run test:unit -- --run → build → playwright install → npm run test:e2e
- Use `npm run check` (svelte-check) for typecheck
- Cache node_modules with `actions/cache` or `setup-node cache: 'npm'`
- Trigger on: `push` to all branches, `pull_request` to main
- Keep E2E step separate from unit step for clear failure isolation
- Dev server must be started before E2E: use `npx wait-on` or start-server pattern

## Out of Scope

- Visual regression (already handled by `visual-regression.yml`)
- Deploy on merge (future sprint)

## Dependencies

## Implementation Notes

**Date:** 2026-03-09
**Files created:**
- `.github/workflows/ci.yml` — main CI pipeline

**Notable decisions:**
- E2E step installs only `chromium` (not all browsers) to reduce CI time
- No separate build step needed — `playwright.config.ts` already runs `npm run build && npm run preview` via `webServer` config
- `svelte-kit sync` is called automatically by `npm run check`, no extra step needed
- Trigger: push to all branches + PR to main (catches feature branches early)

**Known limitations:**
- E2E build happens inside Playwright webServer, so build errors surface during E2E step (not separately — acceptable for now)
