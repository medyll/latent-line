# ST-010 — Bundle Analysis and Size Optimization

**Epic:** Performance
**Sprint:** 2
**Points:** 1
**Priority:** Could
**Status:** Not Started

## User Story

As a developer, I want a production bundle analysis so that I can identify and eliminate unnecessary size from the app before it grows further.

## Acceptance Criteria

```gherkin
Given pnpm run build completes successfully
When the bundle size report is generated
Then it is saved to bmad/artifacts/bundle-analysis-{date}.md

Given the bundle is analyzed
When any single dependency exceeds 50KB gzipped
Then it is flagged in the report with a mitigation suggestion

Given the current bundle baseline is ~450KB (dev)
When the production build is measured
Then the report documents the gzipped total and largest chunks
```

## Technical Notes

- `pnpm run build` produces the Vite output
- Use `vite-bundle-visualizer` or parse `.vite/stats.json` if available
- Alternatively: `du -sh dist/` for quick size check, supplemented by manual chunk inspection
- Document findings in `bmad/artifacts/bundle-analysis-2026-03-12.md`

## Out of Scope

- Implementing tree-shaking changes (flagging only)

## Definition of Done

- [ ] `pnpm run build` succeeds with 0 errors
- [ ] Bundle report artifact created at `bmad/artifacts/bundle-analysis-2026-03-12.md`
- [ ] Total gzipped size documented
- [ ] Any chunk > 50KB gzipped flagged with recommended action
