# Story S28-03: ImportModal — JSON/YAML Import with Validation

**Sprint:** 28  
**Points:** 4  
**Status:** Pending  
**Priority:** High

---

## Description

Build an ImportModal component that allows users to import timeline models from JSON or YAML files, with full validation against the Zod schema.

This completes the import/export round-trip started in S28-01/S28-02.

---

## User Stories

### As a user, I want to:
- Import a `.json` or `.yaml` timeline file
- See validation errors if the file is invalid
- Preview the imported model before confirming
- Merge or replace the current timeline

---

## Acceptance Criteria

### Core Functionality
- [ ] Import button in ExportModal or SystemFooter
- [ ] File picker accepts `.json`, `.yaml`, `.yml`
- [ ] Drag & drop support for file import
- [ ] Validation against `modelSchema` (Zod)
- [ ] Error display with line numbers if possible
- [ ] Preview panel showing imported model summary:
  - Project name, FPS, resolution
  - Asset counts (characters, environments, audio)
  - Timeline event count
  - Total duration

### Import Modes
- [ ] **Replace**: Clear current model, load imported
- [ ] **Merge**: Append timeline events, merge assets
- [ ] **Cancel**: Abort import

### Error Handling
- [ ] Invalid JSON/YAML syntax → show parse error
- [ ] Schema validation failure → show field errors
- [ ] Missing required fields → highlight missing
- [ ] File too large → warn about performance

### Accessibility
- [ ] Keyboard navigation (Tab, Enter, Escape)
- [ ] ARIA labels for file picker
- [ ] Focus trap in modal
- [ ] Screen reader announcements for errors

### Testing
- [ ] Unit tests (10+ tests)
  - Valid JSON import
  - Valid YAML import
  - Invalid JSON error
  - Schema validation errors
  - Merge vs replace modes
- [ ] E2E test (2+ scenarios)
  - Import valid file
  - Import invalid file

```

### Modified Components

```

### Utilities

```
src/lib/utils/import-parser.ts               (~100 lines)
├── parseImportFile(text, fileType) → Model | ParseError
├── validateImportFile(model) → ValidationResult
└── mergeModels(target, source) → Model
```

### Store Updates

```
src/lib/model/model-store.svelte.ts
├── importModel(model: Model, mode: 'replace' | 'merge')
└── clearImportErrors()
```
