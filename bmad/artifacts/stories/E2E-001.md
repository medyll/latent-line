# E2E-001 — Expand E2E tests for asset CRUD and event editing flows

**Epic:** Testing / E2E
**Sprint:** 8
**Points:** 3
**Priority:** Must

## User Story

As a **tester**, I want comprehensive E2E tests for the main user workflows (asset management and event editing), so that core interactions are protected against regressions.

## Context

Sprint 7 deferred E2E test expansion. Current E2E suite (6 tests) covers: app boot, timeline load, home page, and one event-select scenario. Key user flows — asset CRUD, character editing, event property editing — are untested end-to-end.

## Acceptance Criteria

```gherkin
Given the app is loaded at /timeline
When a user adds a new asset via AssetManager
Then the asset appears in the asset list

Given an asset exists in the AssetManager
When a user edits the asset name inline
Then the new name is persisted and displayed

Given the timeline page is loaded
When a user clicks an event in the temporal sequencer
Then the PropertiesPanel shows that event's properties

Given a timeline event is selected
When a user edits a field in PropertiesPanel (e.g., speech text)
Then the change is reflected in the event data

Given the timeline has multiple audio tracks
When the mute button is clicked on a track
Then that track shows the muted state visually

Given the synoptic view and temporal sequencer are both visible
When the synoptic view is scrolled
Then the temporal sequencer scrolls to the same position
```

## Technical Notes

- Add tests to `e2e/assets.spec.ts` and `e2e/event-editing.spec.ts` (new files)
- Use `data-test-id` attributes for selectors (already present on key components)
- Use `page.waitForSelector` with explicit timeouts for SvelteKit hydration
- Target: 6 new E2E scenarios → total E2E count goes from 6 → 12+
- Avoid testing implementation details; test user-observable outcomes only
- Use `expect(locator).toBeVisible()` and `expect(locator).toHaveText()` patterns

## Out of Scope

- Testing dark mode visuals (VR-001)
- Performance timing assertions
- Mobile viewport tests

## Dependencies

- None
