# S22-01 — Infrastructure i18n

**Sprint:** 22
**Points:** 5
**Priorité:** 🟡 Moyenne

## Contexte

L'app est entièrement en anglais hardcodé. Pour toucher un public plus large (notamment francophone), l'internationalisation est nécessaire.

## Objectif

Mettre en place l'infrastructure i18n avec support FR/EN dès le départ.

## Critères d'acceptance

- [ ] Librairie `paraglide-js` (SvelteKit-native, tree-shakeable, 0 runtime overhead)
- [ ] Toutes les chaînes UI extraites dans `src/lib/i18n/messages/en.json` et `fr.json`
- [ ] Détection automatique de la locale via `navigator.language`
- [ ] Fallback sur EN si locale non supportée
- [ ] Variables interpolées : `t('events.count', { count: 5 })` → "5 events"
- [ ] Pluralisation : `t('events', { count: 1 })` → "1 event" / "2 events"

## Implémentation suggérée

- `npm install @inlang/paraglide-js @inlang/paraglide-sveltekit`
- Config `project.inlang` à la racine
- Script de migration : extraction automatique des chaînes hardcodées
