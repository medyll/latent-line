# TST-001 — Screenshot test baseline for all components

- sprint: 7
- created: 20260309T
- status: ready
- points: 3

## Description

Capture baseline screenshots of all major components in the latent-line UI. These serve as reference images for visual regression detection in TST-002.

## Acceptance Criteria

- Baseline screenshots captured for 14 components:
  - SequenceOrchestrator, Synoptic View, Temporal Sequencer
  - AudioTimeline, PropertiesPanel, CharacterField
  - AssetManager, SystemFooter, timeline-selection
  - All 3 cards in empty state
- Screenshots saved to `.github/screenshots/baseline/`
- Document capture resolution (e.g., 1920x1080) and browser (e.g., Chrome)
- Baseline includes both light and dark modes (if theme support exists)
- Screenshots are human-reviewed for correctness (no glitches/clipping)

## Implementation Notes

- Use Playwright `page.screenshot()` with consistent viewport size
- Save as `.png` files named `{ComponentName}.png`
- Create utility script `scripts/capture-screenshots.ts` for reusability
- Document which properties panel states are captured (event selected, asset selected, empty)
- Include at least 3 variations per component if state-dependent

## Out of Scope

- Waveform visual accuracy (deferred to future)
- Dark mode implementation (if not already present)

## Dependencies

- None (can run independently)

## Definition of Done

- [ ] All 14 baseline screenshots captured and committed
- [ ] Screenshot directory structure documented in README
- [ ] Capture script tested locally and works on CI
- [ ] Screenshots human-reviewed for visual correctness
