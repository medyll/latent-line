# TST-001 — Screenshot test baseline for all components

- sprint: 7
- created: 20260309T
- status: ready
- points: 3

## Description

Capture baseline screenshots of all major components in the latent-line UI. These serve as reference images for visual regression detection in TST-002.

## Acceptance Criteria

- Baseline screenshots captured for 14 components:
  - SequenceOrchestrator, Synoptic View, Temporal Sequencer
  - AudioTimeline, PropertiesPanel, CharacterField
  - AssetManager, SystemFooter, timeline-selection
  - All 3 cards in empty state
- Screenshots saved to `.github/screenshots/baseline/`
- Document capture resolution (e.g., 1920x1080) and browser (e.g., Chrome)
- Baseline includes both light and dark modes (if theme support exists)
- Screenshots are human-reviewed for correctness (no glitches/clipping)

## Implementation Notes

- Use Playwright `page.screenshot()` with consistent viewport size
- Save as `.png` files named `{ComponentName}.png`
- Create utility script `scripts/capture-screenshots.ts` for reusability
- Document which properties panel states are captured (event selected, asset selected, empty)
- Include at least 3 variations per component if state-dependent

## Out of Scope

- Waveform visual accuracy (deferred to future)
- Dark mode implementation (if not already present)

## Dependencies

- None (can run independently)

- `scripts/screenshot-list.json` — Configurable list of components/URLs/selectors to capture
- `package.json` — new npm script `capture:screenshots`

**What was delivered:**

1. ✅ Playwright capture utility that can start dev server if needed, capture light/dark variants, and write PNGs to `.github/screenshots/baseline/` (mode: baseline) or `.github/screenshots/current/` (mode: current).
2. ✅ Config-driven list for component captures (`scripts/screenshot-list.json`)
3. ✅ npm script `pnpm run capture:screenshots` for easy invocation

**How to capture baselines (automated):**

1. From project root, run `pnpm run capture:screenshots baseline` (this will start dev server if not running, capture screenshots, then stop the server).
2. Verify images in `.github/screenshots/baseline/`.
3. If any selectors/URLs need adjustment, update `scripts/screenshot-list.json` and re-run.

**Notes / Limitations**

- The default `scripts/screenshot-list.json` uses placeholder URLs (mostly `/timeline`) — review and refine URLs/selectors for each component before running in CI.
- The script assumes Vite dev server on `http://localhost:5173`. Use `BASE_URL` env var to override.

**Next step:** Review captured baselines, human-review for correctness, then commit images to complete TST-001.
