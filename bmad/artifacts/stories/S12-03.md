# S12-03 — Import projet JSON

**Sprint:** 12
**Points:** 3
**Priorité:** 🔴 Critique

## Contexte

Sans import, impossible de reprendre un projet exporté ou de partager un fichier entre
collaborateurs. Complète S12-02.

## Objectif

Permettre de charger un fichier `.json` de projet et remplacer le modèle courant.

## Critères d'acceptance

- [ ] Bouton "Import JSON" dans le SystemFooter
- [ ] Ouvre un `<input type="file" accept=".json">` natif
- [ ] Le fichier est lu, parsé, et validé contre le schéma `Model`
- [ ] Si valide : remplace le modèle courant et met à jour le localStorage
- [ ] Si invalide : affiche un message d'erreur clair dans le footer (pas une alerte browser)
- [ ] Une confirmation ("Voulez-vous remplacer le projet courant ?") est affichée si le modèle actuel a des données
- [ ] Tests unitaires sur la validation du schéma importé

## Implémentation suggérée

- Utiliser `FileReader.readAsText()` + `JSON.parse()`
- Validation légère : vérifier la présence des clés `assets`, `timeline`, `config`
- En cas d'erreur de schéma, lister les champs manquants dans le message
