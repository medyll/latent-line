# S10-03 — Import Timeline JSON

**Sprint**: 10 | **Points**: 2 | **Status**: done ✅ (already implemented in SystemFooter.svelte)

## Description

As a content creator, I can load a previously exported timeline JSON file to replace the current model, so I can resume work across sessions.

## Acceptance Criteria

- An "Import JSON" button (adjacent to Export) opens a file picker (`.json` only)
- On file selection, the file is parsed and validated with `modelSchema.safeParse()`
- If valid: the model store is replaced and the UI re-renders
- If invalid: a user-visible error message is shown (inline, no alert())
- Import does not reload the page

## Technical Notes

- Use `<input type="file" accept=".json">` hidden input + programmatic `.click()`
- Read file with `FileReader.readAsText` → JSON.parse → Zod safeParse
- On success: call a setter passed from the page (e.g. `onImport(newModel)`)
- Error state: a `$state importError: string | null` displayed below the button
