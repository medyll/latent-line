# S11-03 — CSS Audit post-Tailwind

**Sprint**: 11 | **Points**: 3 | **Status**: done ✅

## Description

As a developer, I want all components to render correctly using only the custom CSS stack (theme.css, base.css, workspace.css, utilities.css), with no dependency on Tailwind CSS.

## Acceptance Criteria

- `grep -r "tailwind\|@apply\|tw-" src/` returns no matches
- All components using Tailwind-style class names (flex, gap-\*, text-xs, etc.) are covered by utilities.css
- Any missing utility classes are added to utilities.css
- No broken layouts or invisible elements in the main app view
- `npm run build` exits 0

## Technical Notes

- Sprint 9 removed Tailwind but some components (SequenceOrchestrator, SystemFooter, AudioTimeline) still use Tailwind-like class names — these should be covered by utilities.css
- Check utilities.css for gaps: common classes used but not defined
- If a class is truly unused, remove it; if needed, add the definition
- Do NOT reintroduce Tailwind as a dependency
