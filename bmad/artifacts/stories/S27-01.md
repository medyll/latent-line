# S27-01: Post-Release Bug Triage & Critical Fixes (3 pts)

**Sprint:** 27
**Status:** Ready for planning
**Points:** 3

## Description

Collect and address bugs from v0.4.0 release. Prioritize high-impact issues (data loss, crashes, model corruption) and document workarounds for known limitations. Establish post-release support baseline.

## Acceptance Criteria

1. **Issue Triage**
   - [ ] Review GitHub Issues (if posted after 2026-03-25 release)
   - [ ] Categorize by severity: Critical (crash/data loss), High (blocking feature), Medium (UX friction), Low (cosmetic)
   - [ ] Document discovered issues in `bmad/artifacts/bug-triage-2026-03-XX.md`

2. **Critical Bug Fixes**
   - [ ] Fix any crashes or data corruption issues (if found)
   - [ ] Fix model validation failures that block timeline creation
   - [ ] Fix ComfyUI webhook failures (if any reported)
   - [ ] Add defensive tests for each bug fixed

3. **High-Impact Fixes**
   - [ ] Fix UI layout issues (if reported from v0.4.0)
   - [ ] Fix performance regressions on specific hardware
   - [ ] Fix dark mode toggle on Safari (if issue exists)

4. **Documentation**
   - [ ] Update README with known limitations and workarounds
   - [ ] Document ComfyUI setup troubleshooting
   - [ ] Add FAQ section if patterns emerge

5. **Testing**
   - [ ] All 430+ tests still passing
   - [ ] New tests added for each bug fixed
   - [ ] E2E smoke tests cover fixed issues

## Technical Notes

- Post-release bugs often surface in real-world usage patterns not covered by dev tests
- Prioritize crashes and data loss above all else
- Consider adding error tracking (Sentry or similar) for future releases if not already implemented
- Review ComfyUI webhook error handling — common integration pain point

## Definition of Done

- [ ] All critical bugs fixed
- [ ] Bug triage report generated and documented
- [ ] Tests added for bug fixes
- [ ] README/docs updated with workarounds
- [ ] No new regressions introduced
- [ ] PR merged to main
- [ ] v0.4.1 release (if critical fixes warrant it)
