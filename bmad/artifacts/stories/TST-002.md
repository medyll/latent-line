# TST-002 — Visual regression detection setup (baseline + diff)

- sprint: 7
- created: 20260309T
- status: ready
- points: 3

## Description

Set up automated visual regression detection comparing current screenshots against baselines from TST-001. Generates visual diff reports to catch unintended UI changes.

## Acceptance Criteria

- Regression detection script compares baseline vs. current screenshots
- Diff images generated for any visual changes (pixel-level comparison with threshold)
- Threshold configurable (default: 2% pixel difference tolerance)
- HTML diff report generated showing:
  - Side-by-side baseline vs. current for each component
  - Highlighted regions of change
  - Pass/fail status
- Script runnable locally: `npm run test:visual`
- CI integration ready (can run in GitHub Actions workflow)
- False positive rate < 5% (test with minor code comments, spacing changes)

## Implementation Notes

- Use `pixelmatch` or `jimp` for pixel-level image comparison
- Generate diffs as overlay images (red highlighting changes)
- Store diffs in `.github/screenshots/diffs/` (git-ignored)
- Report format: `visual-regression-report.html`
- Add baseline update script: `npm run test:visual:update` (refreshes baselines after intentional changes)
- Integrate into CI as separate step (allow PR to merge after manual visual review)

## Out of Scope

- Automated screenshot capture on CI (manual baseline refresh for now)
- Cross-browser visual testing (Chrome only initially)

## Dependencies

- Requires: TST-001 (baseline screenshots)

## Definition of Done

- [ ] Visual regression script working locally
- [ ] Tested with intentional UI changes (confirms detection works)
- [ ] CI integration ready and documented
- [ ] Baseline update process documented
- [ ] False positive rate tested and acceptable
