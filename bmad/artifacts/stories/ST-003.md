# ST-003 — PropertiesPanel State Sync

**Sprint:** 1
**Status:** Done
**Priority:** Medium

---

## Description

PropertiesPanel should sync with the global selection state — both timeline events and asset selections — and display sections more cleanly (skip null/empty sections).

## Acceptance Criteria

- [x] Panel accepts `selectedAssetId` prop and shows a selection indicator when an asset is active
- [x] Panel clears event context when an asset is selected (asset selection takes focus)
- [x] Null/undefined sections (camera, lighting, fx, controlnet) are skipped rather than shown as `null`
- [x] Panel title reflects the current selection context (event vs asset)
- [x] timeline.svelte passes `selectedAssetId` to PropertiesPanel

## Technical Notes

- `assetStore` lives inside AssetManager — not available to PropertiesPanel without refactor
- ST-003 shows asset type/id from the `selectedAssetId` string (`type:id` format) without full data
- Full asset data in PropertiesPanel requires lifting `assetStore` to a shared context (deferred to a later story)

## Known Limitations

- PropertiesPanel cannot show full asset properties (name, prompt, etc.) until assetStore is lifted to parent
- Editing timeline frame properties (camera zoom, etc.) is deferred to Sprint 2

## Implementation Notes

**Date:** 2026-03-05
**Files changed:**

- `src/lib/components/app/PropertiesPanel.svelte` — added selectedAssetId prop, skips null sections, shows asset selection context
- `src/lib/components/app/timeline.svelte` — passes selectedAssetId to PropertiesPanel
