# Story S29-01: Virtual Scrolling for Large Timelines

**Sprint:** 29  
**Points:** 5  
**Status:** In Progress  
**Priority:** P0

---

## Description

Implement virtual scrolling (windowing) for the timeline to handle 500-1000+ events without performance degradation.

**Current Problem:**
- Timeline renders ALL events in DOM
- 100+ events = noticeable lag
- 500+ events = unusable

**Target:**
- 1000+ events at 60fps
- Memory usage under 200MB
- Smooth scrolling

---

## Technical Approach

### Library: @tanstack/virtual

Proven, maintained, framework-agnostic virtualization.

```bash
pnpm add @tanstack/virtual
```

### Implementation

```typescript
// src/lib/components/workspace/virtual-timeline.svelte
import { useVirtualizer } from '@tanstack/virtual';

const virtualizer = useVirtualizer({
  count: timeline.length,
  getScrollElement: () => scrollContainer,
  estimateSize: () => 80, // Estimated height per event card
  overscan: 3 // Render 3 items before/after viewport
});

const virtualRows = $derived(virtualizer.getVirtualItems());
```

### Component Structure

```
VirtualTimeline.svelte
├── Scroll container (native scroll)
├── Virtual spacer (maintains scroll height)
└── Virtual items only (windowed)
    └── TimelineEvent (recycled)
```
