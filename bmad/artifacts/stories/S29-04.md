# Story S29-04: UI/UX Polish — Professional, Sober, Balanced Design

**Sprint:** 29  
**Points:** 3  
**Status:** Pending  
**Priority:** High (Quality Gate)

---

## Description

Audit and refine the entire application UI to ensure it meets professional standards:

- **Professionnelle** — Clean, modern, suitable for production use
- **Sobre** — No visual clutter, minimal distractions, focused on content
- **Légère** — Light visual weight, subtle shadows, minimal borders
- **Aérée** — Generous whitespace, comfortable padding, breathable layouts
- **Cohérente** — Consistent spacing, colors, typography across all components
- **Équilibrée** — Visual hierarchy, proportional sizing, harmonious ratios

---

## UX/UI Principles

### 1. Professional (Pro)
- Neutral color palette (grays, whites, subtle accents)
- No decorative gradients or effects
- Functional animations only (no gratuitous motion)
- Clear information hierarchy

### 2. Sober (Sobre)
- Minimal chrome and UI decorations
- Icons used sparingly, only for clarity
- No emojis in production UI
- Subtle states (hover, active, selected)

### 3. Light (Légère)
- Thin borders (1px, subtle colors)
- Light shadows (subtle elevation only)
- Transparent backgrounds where possible
- Visual weight distributed evenly

### 4. Airy (Aérée)
- Generous padding (minimum 12px, typically 16-24px)
- Comfortable line heights (1.5-1.6)
- White space as design element
- No cramped layouts

### 5. Consistent (Cohérente)
- Spacing scale: 4, 8, 12, 16, 24, 32, 48px
- Font sizes: 11px (base), 12px, 14px, 16px, 20px
- Border radius: 4px (small), 8px (medium), 12px (large)
- Colors from design tokens only

### 6. Balanced (Équilibrée)
- Visual weight proportional to importance
- Symmetrical layouts where appropriate
- Golden ratio for key proportions
- Aligned edges and baselines

## Acceptance Criteria

### Visual Quality
- [ ] No layout shifts during interaction
- [ ] All animations under 200ms
- [ ] No visual vibration (competing borders/colors)
- [ ] Scrolling smooth at 60fps

### Consistency
- [ ] All spacing uses design tokens
- [ ] All colors from theme.css
- [ ] All typography from scale
- [ ] No inline styles or magic numbers

### Accessibility
- [ ] All interactive elements 44x44px minimum
- [ ] Focus indicators visible
- [ ] Color not sole indicator of state
- [ ] Text resize to 200% without breaking

### Performance
- [ ] No layout thrashing
- [ ] Paint areas minimized
- [ ] Compositor-only animations
- [ ] First contentful paint < 1.5s

## Technical Approach

### CSS Architecture

```css
/* theme.css — Design tokens */
:root {
  /* Spacing scale */
  --space-1: 0.25rem;   /* 4px */
  --space-2: 0.5rem;    /* 8px */
  --space-3: 0.75rem;   /* 12px */
  --space-4: 1rem;      /* 16px */
  --space-6: 1.5rem;    /* 24px */
  --space-8: 2rem;      /* 32px */
  --space-12: 3rem;     /* 48px */
  
  /* Typography scale */
  --text-xs: 0.6875rem;   /* 11px */
  --text-sm: 0.75rem;     /* 12px */
  --text-base: 0.875rem;  /* 14px */
  --text-lg: 1rem;        /* 16px */
  --text-xl: 1.25rem;     /* 20px */
  
  /* Border radius */
  --radius-sm: 0.25rem;   /* 4px */
  --radius-md: 0.5rem;    /* 8px */
  --radius-lg: 0.75rem;   /* 12px */
  
  /* Shadows — subtle only */
  --shadow-sm: 0 1px 2px rgba(0,0,0,0.04);
  --shadow-md: 0 4px 8px rgba(0,0,0,0.06);
  --shadow-lg: 0 8px 24px rgba(0,0,0,0.08);
}
```

### Refactoring Guidelines

1. **No inline styles** — Use utility classes
2. **No magic numbers** — Use design tokens
3. **No !important** — Fix specificity properly
4. **No hardcoded colors** — Use theme vars
