# ST-020 — AssetManager inline editing + orphan detection

- sprint: 5
- created: 20260308T
- status: done
- points: 5

## Description

Add inline editing to all asset types in AssetManager (AM-02/03). Each row gets an edit toggle (pencil icon) that expands an inline form. Add orphan detection badges (AM-08/09): assets with 0 timeline references show an orange badge.

## Acceptance Criteria

- Character row: edit name, voice_id; outfit list with add/delete (AM-02/03)
- Environment row: edit prompt and ref fields (AM-05)
- Audio row: edit label and url fields (AM-07)
- Assets referenced 0 times in model.timeline show orphan badge
- Reference count shown per character (how many actors use it) and audio asset
- Unit tests for orphan detection logic

## Implementation Notes

- `AssetManager.svelte` fully rewritten with `getContext<Model>(MODEL_STORE_KEY)` for orphan detection
- `editingId = $state<string | null>(null)` toggle controls which row shows inline edit form
- Derived `usedCharIds: Set<string>` and `charRefCounts: Record<string, number>` from `model.timeline`
- Derived `usedAudioIds: Set<string>` and `audioRefCounts: Record<string, number>` from `model.timeline`
- Orange "orphan" badge when ref count = 0; green badge with count when ref count > 0
- Character inline form: name, voice_id inputs; outfit chips with add/delete
- Environment inline form: prompt textarea, ref field
- Audio inline form: label, url fields
- New asset IDs generated with `Date.now()` string for uniqueness
- `AssetManager-orphan.test.ts`: 9 tests covering `computeUsedCharIds`, `computeCharRefCounts`, `computeUsedAudioIds`, `computeAudioRefCounts` — all passing
