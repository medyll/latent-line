# ST-005 — ModelInspector Validation

**Sprint:** 1
**Status:** Done
**Priority:** Medium

---

## Description

Improve ModelInspector validation feedback: fix Svelte 4 class directive, remove redundant inline style, and display Zod issues as a readable field-level list.

## Acceptance Criteria

- [x] `class:success` directive removed — replaced with Tailwind conditional class string
- [x] Redundant inline `style="background: ..."` removed
- [x] `<style>` block removed (was only used for the now-redundant `.success` class)
- [x] Validation errors show per-field issues from `r.error.issues` instead of raw `r.error.message`
- [x] Optional chaining `validation?.ok` replaces `validation && validation.ok` verbosity

## Technical Notes

- In Svelte 5, `class:foo={cond}` still works but `class={cond ? 'foo' : ''}` is the idiomatic approach
- `ZodError.message` concatenates all issues into a single string; `ZodError.issues` is the structured array
- Issue path displayed as `path.join('.')` (e.g. `timeline.0.frame.actors`)

## Implementation Notes

**Date:** 2026-03-05
**Files changed:**

- `src/lib/components/app/ModelInspector.svelte` — fixed class syntax, removed style block, improved validation display

**Notable decisions:**

- Kept manual validation trigger (Validate button) — auto-validation on switch adds noise when template model may be intentionally incomplete
- Capped displayed issues at 10 to avoid overwhelming the UI on deeply invalid models
