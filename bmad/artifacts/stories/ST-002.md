# ST-002 — Timeline Event Rendering

**Sprint:** 1
**Status:** Done
**Priority:** High

---

## Description

Implement fully functional timeline event rendering with correct temporal strip positioning and event selection.

## Acceptance Criteria

- [x] TimelineEvent.svelte uses Svelte 5 `$props()` syntax
- [x] Lower temporal strip renders each event at the correct horizontal position proportional to its time value
- [x] Lower temporal strip items are clickable and update `selectedEventId`
- [x] Event durations in the strip span until the next event's start (not a fixed +10 offset)
- [x] Zoom slider correctly scales item widths and positions in the temporal strip

## Technical Notes

- `TimelineEvent.svelte` uses legacy Svelte 4 `export let` — must migrate to `$props()`
- Lower pane uses `left` CSS on flex children (wrong) — container needs `position: relative`, items need `position: absolute`
- Current `end: event.time + 10` produces visual events only 10 frames wide; they should span to next event
- `BASE_PX_PER_FRAME = 1` gives 600px total width at zoom 100 — sensible

## Implementation Notes

**Date:** 2026-03-05
**Files changed:**

- `src/lib/components/app/TimelineEvent.svelte` — migrated to Svelte 5 `$props()`, added `lang="ts"`
- `src/lib/components/app/timeline.svelte` — fixed end times, fixed temporal strip positioning

**Notable decisions:**

- Computed `end` as `nextEvent.time` (or `last.time + 120` for final event) so blocks fill the strip meaningfully
- Used `position: absolute` on strip items inside a `position: relative` scroll container
- `pxPerFrame = BASE_PX_PER_FRAME * (zoom / 100)` keeps zoom semantically consistent

**Known limitations:**

- Temporal strip does not yet support drag-to-reorder events
