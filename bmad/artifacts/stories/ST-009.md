# ST-009 — E2E Tests for Component Interactions

**Epic:** Quality
**Sprint:** 2
**Points:** 3
**Priority:** Should
**Status:** Not Started

## User Story

As a developer, I want E2E tests covering the core component interaction flows so that regressions in asset selection and timeline event selection are caught automatically.

## Context

The existing E2E test only covers the home page h1. This story adds Playwright tests for the timeline route, covering asset click → selection highlight and timeline event click → PropertiesPanel update.

## Acceptance Criteria

```gherkin
Given the /timeline route is open
When I click on a character row in AssetManager
Then that row has the selection highlight class (bg-blue-100)
And PropertiesPanel shows "Character" in its heading

Given the /timeline route is open
When I click on a timeline event card
Then the card has the ring-blue-500 class
And PropertiesPanel shows the event label

Given the /timeline route is open
When I click the same timeline event card again
Then the selection is cleared
And PropertiesPanel shows the empty state message
```

## Technical Notes

- Playwright config already exists (`playwright.config.ts`)
- Add tests to `e2e/timeline.test.ts`
- Use `page.locator('[aria-label="Asset Manager"]')` and `[aria-label="Timeline event Event 1"]`
- `aria-selected` attribute on `role="option"` items enables reliable assertions

## Out of Scope

- Testing CRUD operations (add/remove assets) — deferred to Sprint 3

## Dependencies

- ST-001 (selection attributes in place) ✅
- ST-002 (timeline event cards rendered) ✅

## Definition of Done

- [ ] `e2e/timeline.test.ts` created with 3+ test cases
