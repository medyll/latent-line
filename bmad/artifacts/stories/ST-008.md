# ST-008 — PropertiesPanel: Inline Frame Editing

**Epic:** Editing
**Sprint:** 2
**Points:** 5
**Priority:** Should
**Status:** Not Started

## User Story

As a user, I want to edit camera zoom and lighting type directly from the PropertiesPanel when a timeline event is selected, so that I can tweak frame parameters without manually editing the model JSON.

## Context

PropertiesPanel currently shows raw JSON `<pre>` blocks for camera and lighting. This story replaces those with actual input controls (slider for zoom, select for lighting type) and updates the timeline event data reactively.

## Acceptance Criteria

```gherkin
Given a timeline event is selected
When PropertiesPanel shows Camera section
Then camera.zoom is displayed as a range slider (0.1 – 5.0, step 0.1)

Given the user drags the zoom slider
When they release
Then the timeline event's camera.zoom updates reactively

Given a timeline event is selected
When PropertiesPanel shows Lighting section
Then lighting.type is displayed as a select with all LightingType options

Given the user changes the lighting type
When they confirm selection
Then the timeline event's lighting.type updates reactively

Given a field is updated
When the user re-opens the timeline event card
Then the updated value is reflected in TimelineEvent display
```

## Technical Notes

- `LightingType = 'dusk' | 'daylight' | 'studio' | 'tungsten' | 'ambient'`
- Timeline events in `timeline.svelte` are derived from `exampleModel` — to make them editable, convert `timelineEvents` to a `$state` array so mutations propagate
- PropertiesPanel receives the `timelineEvents` array and mutates items by index/id
- Use immutable update: `timelineEvents[idx] = { ...ev, frame: { ...ev.frame, camera: { ...ev.frame.camera, zoom: newZoom } } }`
- FX and audio_tracks remain as JSON for now (scoped to camera + lighting only)

## Out of Scope

- Editing actor positions, speech, or outfit
- Undo/redo
- Persisting changes
