# S25-01 — Export EDL (Edit Decision List)

**Sprint:** 25
**Points:** 8
**Priorité:** 🟡 Moyenne

## Contexte

Les équipes de post-production utilisent des EDL pour importer des séquences dans DaVinci Resolve, Premiere Pro ou Final Cut. Un export EDL permettrait d'intégrer latent-line dans le workflow professionnel.

## Objectif

Exporter la timeline au format EDL standard (CMX 3600).

## Critères d'acceptance

- [ ] Export "EDL" dans le menu Export
- [ ] Format CMX 3600 compatible DaVinci Resolve et Premiere Pro
- [ ] Chaque event → une entrée EDL avec timecode SMPTE (basé sur FPS du projet)
- [ ] Le nom de l'event comme clip name dans l'EDL
- [ ] Validation du timecode : pas de chevauchement, pas de gap > 1 frame
- [ ] Tests unitaires avec fixture EDL de référence

## Implémentation suggérée

- `src/lib/utils/export-edl.ts`
- Conversion frame → SMPTE : `frameToTimecode(frame, fps)`
- Spec CMX 3600 implémentée selon la documentation publique
