# S17-03 — Drag & drop pour réordonner les events

**Sprint:** 17
**Points:** 8
**Priorité:** 🟠 Haute

## Contexte

Les events ne peuvent être réordonnés que via suppression/recréation. Le drag & drop est une attente UX de base pour une timeline.

## Objectif

Permettre de déplacer les events par drag & drop dans la synoptic view et le temporal sequencer.

## Critères d'acceptance

- [ ] Drag d'un event card dans la synoptic view → réordonne dans la liste
- [ ] Drag d'un event dans le temporal sequencer → déplace le time de début
- [ ] Indicateur de drop zone (ligne bleue entre cards)
- [ ] Snap au frame le plus proche dans le temporal sequencer
- [ ] L'opération est undoable (Ctrl+Z)
- [ ] Touch support (pointer events, pas mouse events)

## Implémentation suggérée

- API native Drag & Drop HTML5 ou Pointer Events
- Action Svelte `use:draggable` / `use:droppable`
- Pas de dépendance externe (garder le bundle léger)
