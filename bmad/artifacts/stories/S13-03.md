# S13-03 — Recherche et filtre dans AssetManager

**Sprint:** 13
**Points:** 3
**Priorité:** 🟡 Moyenne

## Contexte

L'AssetManager n'a pas de recherche. Avec un projet de 20+ assets, retrouver un asset
spécifique devient laborieux.

## Objectif

Ajouter un champ de recherche et des filtres par type dans l'AssetManager.

## Critères d'acceptance

- [ ] Champ de recherche texte en haut de l'AssetManager (filtre en temps réel)
- [ ] La recherche porte sur le nom de l'asset (insensible à la casse)
- [ ] Filtres par type : All / Characters / Environments / Audio (tabs ou boutons toggle)
- [ ] Le filtre actif est visuellement mis en évidence
- [ ] Si aucun résultat : message "Aucun asset trouvé"
- [ ] La recherche et le filtre sont combinables
- [ ] L'état de recherche est local au composant (pas persisté)
- [ ] Tests unitaires sur la logique de filtrage

## Implémentation suggérée

- `$state` local `searchQuery` + `activeFilter` dans AssetManager.svelte
- `$derived` pour la liste filtrée
- Input avec `oninput` pour mise à jour réactive
