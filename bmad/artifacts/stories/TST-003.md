# TST-003 — Expand model schema edge case tests

- sprint: 7
- created: 20260309T
- status: ready
- points: 2

## Description

Add comprehensive unit tests for model validation covering edge cases, boundary conditions, and invalid inputs to ensure robustness of TimelineFrame and Config schemas.

## Acceptance Criteria

- Edge case tests for TimelineFrame:
  - Empty actors array, null/undefined fields
  - Maximum array sizes (100+ events in timeline)
  - Negative numbers (time, pan values, zoom < 0.1)
  - Float precision (zoom 1.25, tilt 45.5)
- Edge case tests for Config:
  - audioLanes with duplicate IDs
  - Solo + mute both true simultaneously
  - Seed boundaries (0, 999999, -1, 1000000)
- Edge case tests for Character/AudioAsset:
  - Empty string names/IDs
  - Very long text (>1000 chars)
  - Special characters in names (emoji, unicode)
- Unit test count: +15 tests minimum
- Test coverage for model-types.ts: >= 85%

## Implementation Notes

- Add tests to `src/lib/model/model-template.test.ts`
- Use `describe('Edge Cases', ...)` for organization
- Test both valid edge cases (should pass) and invalid ones (should fail validation)
- Document expected behavior for each edge case
- Use Zod `.safeParse()` to verify validation results

## Out of Scope

- Performance tests (load testing with 10k+ events)
- Integration tests with actual audio files

## Dependencies

- None (uses existing test infrastructure)

## Definition of Done

- [ ] 15+ new edge case tests added
- [ ] All tests passing
- [ ] Model validation coverage >= 85%
- [ ] Edge cases documented in test comments
