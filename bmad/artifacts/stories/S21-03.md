# S21-03 — Layout responsive (tablet / mobile)

**Sprint:** 21
**Points:** 5
**Priorité:** 🟡 Moyenne

## Contexte

L'app est conçue pour desktop uniquement. Sur tablette, les panneaux sont trop étroits. Sur mobile, l'app est inutilisable.

## Objectif

Rendre l'app utilisable sur tablette (1024px) et navigable sur mobile (375px).

## Critères d'acceptance

- [ ] Breakpoint 1024px : layout 2 colonnes (AssetManager + workspace), PropertiesPanel en panneau coulissant
- [ ] Breakpoint 768px : layout 1 colonne avec tabs (Assets | Timeline | Properties)
- [ ] Breakpoint 375px : synoptic view en mode liste compacte, temporal sequencer masqué
- [ ] Swipe horizontal pour changer d'onglet sur mobile
- [ ] Toutes les fonctions CRUD accessibles sur tous les breakpoints

## Implémentation suggérée

- CSS custom properties `--layout-mode: desktop|tablet|mobile` via JS `matchMedia`
- Navigation onglets mobile : `TabBar.svelte`
- Panneau coulissant : `SlidePanel.svelte` avec `transform: translateX`
