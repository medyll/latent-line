# S12-02 — Export projet JSON

**Sprint:** 12
**Points:** 3
**Priorité:** 🔴 Critique

## Contexte

Aucun mécanisme d'export n'existe. L'utilisateur ne peut pas sauvegarder son travail
sous forme de fichier portable.

## Objectif

Permettre d'exporter le modèle courant en fichier `.json` téléchargeable.

## Critères d'acceptance

- [ ] Bouton "Export JSON" visible dans le SystemFooter (ou menu fichier)
- [ ] Le clic génère un `Blob` JSON et déclenche un download nommé `{projectName}-{date}.json`
- [ ] Le JSON exporté respecte exactement le schéma `Model` (model-types.ts)
- [ ] Le fichier est lisible et ré-importable (voir S12-03)
- [ ] Test unitaire sur la fonction d'export

## Implémentation suggérée

```ts
function exportProject(model: Model) {
  const json = JSON.stringify(model, null, 2)
  const blob = new Blob([json], { type: 'application/json' })
  const url = URL.createObjectURL(blob)
  const a = document.createElement('a')
  a.href = url
  a.download = `${model.config.name ?? 'project'}-${Date.now()}.json`
  a.click()
  URL.revokeObjectURL(url)
}
```
