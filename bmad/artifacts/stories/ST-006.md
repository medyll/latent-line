# ST-006 — Lift assetStore to Svelte Context

**Epic:** Shared State
**Sprint:** 2
**Points:** 3
**Priority:** Must
**Status:** Done

## User Story

As a developer, I want the assetStore to be available via Svelte context so that both AssetManager and PropertiesPanel can read and write the same asset data.

## Context

Currently `assetStore` is a local `$state` inside `AssetManager.svelte`. This prevents PropertiesPanel from showing asset details when an asset is selected. Moving it to a shared Svelte context (via `setContext`/`getContext`) makes it available across the component tree without prop-drilling.

## Acceptance Criteria

```gherkin
Given the app is mounted
When any component calls getContext('assetStore')
Then it receives the reactive asset store instance

Given AssetManager adds a new character
When PropertiesPanel reads the same context
Then it sees the updated character list

Given AssetManager is rendered inside the app layout
When the context is set in a common ancestor (e.g. app.svelte or app-content.svelte)
Then both AssetManager and PropertiesPanel can access it
```

## Technical Notes

- Use `setContext('assetStore', assetStore)` in the nearest common ancestor of AssetManager and PropertiesPanel
- `assetStore` remains `$state<Assets>(structuredClone(exampleModel.assets))`
- AssetManager reads/writes via `getContext('assetStore')` instead of local state
- PropertiesPanel reads via `getContext('assetStore')` (read-only)
- Svelte context key: use a Symbol exported from a shared module (e.g. `src/lib/context/keys.ts`) to avoid string collisions

## Out of Scope

- Persisting assetStore to localStorage or a backend
- Multiple simultaneous asset stores

## Dependencies

- None (first story of the sprint)
