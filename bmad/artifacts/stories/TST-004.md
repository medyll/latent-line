- sprint: 7

## Description

Ensure all user-facing input fields (forms, selectors, text areas) validate correctly and reject malicious or malformed input. Test sanitization, type coercion, and error messages.

## Acceptance Criteria

- Input validation tests for:
  - Checkpoint/Sampler/TTS selectors (invalid string rejection)
  - Seed input (non-numeric, overflow)
  - Speech text (XSS prevention via sanitizeText)
  - Camera pan/zoom (out-of-range rejection)
  - Character name field (length limits, special chars)
  - Mood selector (invalid enum values)
- Validation error messages clear and user-friendly
- Type coercion tests (string "1.5" → number 1.5)
- Security tests: confirm XSS payloads are sanitized
- Test count: +12 validation tests
- All existing tests still passing after changes

## Implementation Notes

- Add tests to `src/lib/components/app/PropertiesPanel.test.ts` and related component test files
- Test both Zod schema validation and component-level input handlers
- Use fixtures for common invalid inputs (null, undefined, empty string, NaN, Infinity)
- Document validation rules for each field in component tests
- Verify sanitizeText() prevents common XSS vectors

## Out of Scope

- Accessibility validation (ARIA labels already tested in existing tests)
- Database injection tests (no backend in MVP)
