# S13-01 — Undo / Redo

**Sprint:** 13
**Points:** 8
**Priorité:** 🟠 Haute

## Contexte

Toutes les mutations du modèle sont directes et irréversibles. Une suppression accidentelle
d'un event ou d'un asset détruit le travail sans recours.

## Objectif

Implémenter un historique des actions avec undo (Ctrl+Z) et redo (Ctrl+Y).

## Critères d'acceptance

- [ ] Ctrl+Z annule la dernière action (jusqu'à 50 niveaux)
- [ ] Ctrl+Y / Ctrl+Shift+Z remet l'action annulée
- [ ] Les boutons ↩ / ↪ sont affichés dans la topbar avec état disabled si pas d'historique
- [ ] Actions couvertes : add/delete/duplicate event, edit event property, add/delete/edit asset
- [ ] L'historique est réinitialisé lors d'un import ou "New project"
- [ ] Tests unitaires sur le stack undo/redo

## Implémentation suggérée

- Pattern Command : chaque mutation produit un objet `{ do: () => void, undo: () => void }`
- Store `historyStore` avec deux stacks : `past[]` et `future[]`
- Wrapping des setters existants du model store
- Snapshot immutable via `structuredClone` ou `JSON.parse/stringify`

## Risque

- Complexité élevée si les setters actuels sont des mutations directes Svelte 5 `$state`
- Option alternative plus simple : snapshots complets du modèle (moins élégant mais robuste)
