# CI-002 — README CI badges + test coverage reporting

**Epic:** CI / Documentation
**Sprint:** 8
**Points:** 2
**Priority:** Should

## User Story

As a **developer or contributor**, I want CI status badges in the README and a test coverage report, so that the project's health is visible at a glance.

## Context

CI-001 establishes the pipeline. CI-002 surfaces CI results: a badge in the README for the main CI workflow, and a vitest coverage report to quantify the 218+ tests.

## Acceptance Criteria

```gherkin
Given the CI pipeline is green
When a developer visits the GitHub repository
Then the README shows a green CI badge linking to the Actions run

Given unit tests run with coverage enabled
When the CI pipeline completes
Then a coverage summary is available (Istanbul/v8 report)

Given a PR is opened
When CI runs
Then coverage delta is visible (via codecov or inline report)
```

## Technical Notes

- Add badge to `README.md`: `![CI](https://github.com/<owner>/latent-line/actions/workflows/ci.yml/badge.svg)`
- Enable vitest coverage: add `--coverage` to `test:unit` script or add `coverage` config in `vitest.config.ts`
- Use `@vitest/coverage-v8` (already compatible with vitest 4)
- Coverage output: `lcov` format → upload as CI artifact or to codecov (optional)
- Add `coverage` to `.gitignore` if not already present
- Script: `"test:unit:coverage": "vitest run --coverage"` in package.json

## Out of Scope

- Branch protection rules requiring CI pass (GitHub repo settings — manual)
- Code coverage enforcement threshold (future sprint)

## Dependencies

- CI-001 must be complete (badge requires the workflow file to exist)
