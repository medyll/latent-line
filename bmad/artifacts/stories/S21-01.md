# S21-01 — Animations et transitions d'état

**Sprint:** 21
**Points:** 3
**Priorité:** 🟡 Moyenne

## Contexte

L'UI actuelle est fonctionnelle mais statique. Les changements d'état (sélection, ajout, suppression) sont brutaux et nuisent à la perception de qualité.

## Objectif

Ajouter des transitions CSS/Svelte sur les interactions clés pour une expérience plus fluide.

## Critères d'acceptance

- [ ] Ajout d'un event : slide-in depuis le bas (200ms ease-out)
- [ ] Suppression d'un event : fade-out + collapse (150ms)
- [ ] Sélection d'un event : highlight animé (100ms)
- [ ] Ouverture de modale/panneau : fade + slide (200ms)
- [ ] Transitions respectent `prefers-reduced-motion` (désactivées si demandé)
- [ ] Pas de jank : toutes les animations sur `transform`/`opacity` uniquement

## Implémentation suggérée

- Directives Svelte `transition:fly`, `transition:fade`, `animate:flip`
- `@media (prefers-reduced-motion: reduce)` dans le CSS global
