# ST-001 — AssetManager Interactivity

**Sprint:** 1
**Status:** Done
**Priority:** High

---

## Description

Add asset selection state to AssetManager and fix the `addEnvironment` reactivity no-op.

## Acceptance Criteria

- [x] `addEnvironment` no longer has a spurious `assetStore.environments = assetStore.environments` reassignment
- [x] Clicking any asset row (character, environment, audio) selects it and highlights the row
- [x] Clicking the same row again deselects it (toggle)
- [x] `selectedAssetId` is exposed as a `$bindable` prop so parent components can react to selection
- [x] The delete button does not trigger selection (stopPropagation)
- [x] Unused `EmptyContent` import is removed

## Technical Notes

- Selection key format: `"{type}:{id}"` (e.g. `"char:char_01"`, `"env:dead_city"`, `"audio:low_rumble"`)
- Svelte 5 `$bindable` allows two-way binding from parent: `<AssetManager bind:selectedAssetId />`
- In Svelte 5, `$state` deep proxy handles `assetStore.environments[newId] = {...}` correctly without reassignment

## Implementation Notes

**Date:** 2026-03-05
**Files changed:**

- `src/lib/components/app/AssetManager.svelte` — added selection, fixed addEnvironment, cleaned imports

**Notable decisions:**

- `type:id` key avoids collisions across asset types (characters/envs/audio could share IDs)
- `$bindable` chosen over an `onselect` callback for ergonomic two-way binding in timeline.svelte

**Known limitations:**

- Asset editing (rename, prompt update) is deferred to a later story
