## Objectif

Valider et tester les champs du settings panel (config: checkpoint, sampler, seed, tts_engine).
L'UI existe dans SystemFooter — cette story couvre la validation, les valeurs par défaut, et les tests.

## Acceptance Criteria

- `model.config` accepte les champs optionnels checkpoint, sampler, seed, tts_engine
- seed est un number (pas de string)
- Les champs string vides sont coercés en `undefined` (comportement UI)
- Les audioLanes sont optionnels et validés si présents
- Tests unitaires couvrent : config valide, config partielle, seed hors range, audioLanes

## Status

UI implémentée dans SystemFooter.svelte — tests à écrire.
