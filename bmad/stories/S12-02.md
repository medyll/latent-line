## Objectif

Permettre à l'utilisateur d'exporter le modèle courant en fichier JSON valide.

## Acceptance Criteria

- `exportModel()` génère un Blob JSON via `modelSchema.safeParse()`
- Le fichier téléchargé s'appelle `model.json`
- Si la validation échoue, les erreurs sont affichées (pas de téléchargement)
- Les erreurs Zod sont accessibles via `exportErrors`
- Tests unitaires couvrent : export valide, export avec erreurs, structure du fichier
