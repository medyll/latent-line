Latent-line
vs CapCut AI Studio

Analyse design, fonctionnalités, ambiance et flow — avec recommandations priorisées pour aligner latent-line sur les standards du marché.

Référence marché CapCut AI Studio (mars 2026)
Stack SvelteKit 5 + ComfyUI
Recommandations 17 items · 3 niveaux

🔭

Positionnement comparé

Axe Latent-line CapCut AI Studio
Paradigme Timeline linéaire (séquentiel) Canvas infini spatial (non-linéaire)
Cible Power-user / ComfyUI Creator economy → grand public
Génération Export vers ComfyUI (indirect) Native inline — Seedance 2.0 / Seedream 5.0
Modèle AI Local / ComfyUI / ouvert Propriétaire ByteDance — #1 T2I benchmark
Distribution Export FCPX / Premiere / CogVideo TikTok natif intégré
Onboarding Aucun visible Templates gallery + 150 crédits gratuits
Identité visuelle shadcn-svelte générique Dark dense + AI glow purple/teal
Atout majeur Modèle de données narratif riche, 8+ formats export, collab WebSocket Full-stack créateur → commerce, free tier agressif
Latent-line a une infrastructure technique supérieure à CapCut sur l'édition narrative. Le gap est dans la boucle de feedback visuel et l'ambiance créative — pas dans la profondeur fonctionnelle.

Design

Écart principal : CapCut a un système de signaux visuels clair humain/AI (halo coloré). Latent-line n'en a aucun — zéro différenciation entre contenu généré et contenu édité manuellement.
Design · P1
AI Glow Signal
Chaque frame/event généré par AI doit avoir un halo purple/teal. Pattern industrie émergent (CapCut, Figma AI, Midjourney). Signal visuel immédiat : humain vs AI-generated.
TimelineEvent.svelte
Design · P1
Card visuelle par event
Chaque event timeline = card autonome avec thumbnail de frame (générée ou placeholder avec prompt). CapCut : "card-as-unit" inspiré de Figma/Notion. Unité atomique de création.
TimelineEvent.svelte
Design · P2
Identité chromatique
Tailwind config = 100% CSS vars sans valeurs concrètes. Résultat : palette shadcn gris/blanc générique. Définir une palette sombre avec accent violet/indigo cohérent avec l'univers AI génératif.
tailwind.config.cjs · app.css
Design · P2
Densité adaptative
CapCut : panels denses, canvas aéré. Augmenter gap/padding dans la zone preview, compresser les panels latéraux. Espace respirable = signal de qualité pro.
PropertiesPanel.svelte · PreviewPanel.svelte
Design · P3
Typography distincte
System fonts = aucune personnalité. Ajouter une font mono pour prompts/timecodes (déjà sémantiquement justifié), une sans-serif distinctive pour les headers.
app.css
Design · P3
Mood indicator sur cards
Système de moods présent dans le modèle (joyful, melancholic, anxious…). Ajouter un micro-indicateur coloré par mood sur chaque card timeline. Signal émotionnel immédiat.
TimelineEvent.svelte · model-types.ts
✓ Dark mode via mode-watcher ✓ shadcn-svelte composants ✓ Resizable panels via paneforge ✗ AI glow signal ✗ Thumbnails sur timeline ✗ Identité couleur propre ~ Mood system (modèle OK, UI absente)

Fonctionnalités

Écart principal : Génération native absente. Tout passe par export → ComfyUI → résultat externe. La boucle créative est cassée — c'est le gap le plus douloureux vs CapCut.
Feature · P1
Import résultats ComfyUI → preview inline
comfy-api-client.ts existe déjà. L'étendre pour récupérer les outputs et les afficher dans PreviewPanel.svelte et les cards timeline. Sans ça, pas de boucle créative.
comfy-api-client.ts · PreviewPanel.svelte
Feature · P1
Génération one-click inline
GenerateButton.svelte existe. Le rendre accessible directement depuis chaque card timeline (hover action) sans modal. CapCut : chaque shot promptable en 1 clic inline.
GenerateButton.svelte · TimelineEvent.svelte
Feature · P2
Gallery templates visuels
TemplatesPanel.svelte existe probablement en texte/JSON. Ajouter des thumbnails visuels et un mode "inspiration" avec exemples pré-remplis. Onboarding zero-to-project en 30 secondes.
TemplatesPanel.svelte
Feature · P2
Prompt assist contextuel
PromptAssist.svelte existe. L'enrichir : suggestions basées sur le contexte (mood du projet, personnages existants, scène précédente). CapCut propose de l'aide inline.
PromptAssist.svelte
Feature · P2
Character consistency tracking
Latent-line a le modèle (LoRA, references, outfits) mais aucun indicateur de cohérence entre events. Ajouter warning si personnage change d'outfit entre frames sans transition explicite.
AssetManager.svelte · Timeline.svelte
Feature · P3
Playback flipbook réel
Quand les frames sont importées, faire un vrai flipbook/preview vidéo dans PreviewPanel.svelte. Actuellement scrubbing de timecodes sans visuels.
PreviewPanel.svelte · playback.ts
✓ Export 8+ formats (FCPX, Premiere, CogVideo…) ✓ ComfyUI client API existant ✓ Render queue + batch generate ✓ Collab WebSocket ✗ Import résultats ComfyUI ✗ Preview inline frames ✗ Génération one-click par card ~ Templates (JSON, pas visuel)

Ambiance

Écart principal : Latent-line ressemble à un outil dev. CapCut ressemble à un studio créatif. Le langage, les états vides, les noms de sections — tout signale "développeur" plutôt que "réalisateur".
  • 🎬
    Langage cinéma dans l'UI
    Remplacer la terminologie dev dans l'interface visible : AssetManager → Casting, SequenceOrchestrator → Script, ModelInspector → Régie. Garder les noms techniques dans le code.
  • 🎭
    État vide chargé d'intention
    CapCut : l'état vide propose une galerie d'inspiration + CTA clair. EmptyState.svelte existe. Montrer un exemple de projet finalisé dès l'ouverture — pas un écran blanc.
  • 📻
    Waveform audio réel
    AudioTimeline.svelte existe. Ajouter un waveform SVG sur la piste audio. L'absence de waveform dans un outil vidéo/narrative casse immédiatement l'ambiance studio pro.
  • 🏠
    Landing avec narratif produit
    La route / doit transmettre en 5 secondes : "Tu construis des films AI ici". Pas "tu édites un JSON de timeline". Identité visuelle forte, exemple animé, CTA unique.
  • 📸
    Snapshots comme checkpoints créatifs
    SnapshotsPanel.svelte existe. Les positionner comme des "versions de tournage" (Rough cut v1, Director's cut, Final) — pas comme un backup technique. Label + thumbnail + date.

🔄

Flow utilisateur

Écart principal : CapCut = canvas → générer → publier en 3 étapes. Latent-line = configurer → exporter → aller ailleurs → revenir. La friction est hors-app.
Flow · P1
Fermer la boucle génération
Flow actuel : éditer → exporter → ouvrir ComfyUI → générer → récupérer fichiers → revenir = 5 étapes hors-app. Avec comfy-api-client.ts existant, réduire à : éditer → générer → voir résultat.
Flow · P1
Progression queue visible partout
render-queue.ts + batch-generate.ts existent. Ajouter un indicateur persistant en header : "3/12 frames générées" visible depuis n'importe quelle vue.
render-queue.ts · +layout.svelte
Flow · P2
Onboarding progressif 3 étapes
Zéro onboarding actuellement. Proposition : (1) nommer projet + mood, (2) créer 1 personnage, (3) ajouter 1 event + générer. workflow-builder.ts existant = bon candidat.
workflow-builder.ts
Flow · P2
Presentation mode accessible
Route /present existe mais cachée. Ajouter un bouton "Screening" visible depuis l'app principale. CapCut = publish/share one-click. Latent-line a le feature, pas l'accès.
present/+page.svelte · +layout.svelte
